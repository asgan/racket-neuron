1 Process

 (require neuron/process) package: neuron

A process is a thread-like concurrency abstraction. Under the hood, processes communicate via channel synchronization. Every process has an input channel and an output channel.

A process can be used as a synchronizable event. A process is ready for synchronization when wait would not block. The synchronization result is the process itself.

1.1 Starting and Stopping Processes

procedure

(process thunk    
  [#:on-stop on-stop    
  #:command handler]) → process?
  thunk : (-> any)
  on-stop : (-> any) = void
  handler : (-> any) = void
Calls thunk with no arguments in a new process with command handler handler. If the process terminates gracefully, on-stop will be called with no arguments before the process dies. Returns immediately with a process descriptor value.

Unhandled exceptions are fatal to processes. Attempting to sync on a process killed by an unhandled exception will raise an exception.

procedure

(process? v) → boolean?

  v : any/c
Returns #t if v is a process, #f otherwise.

procedure

(current-process) → process?

Returns the process descriptor for the currently executing process.

procedure

(command π v) → any

  π : process?
  v : any/c
Applies the command handler of π to v inside the calling process and returns the result.

procedure

(quit v ...) → void?

  v : any/c
Gracefully terminates the current process. Each v argument is ignored.

procedure

(die v ...) → void?

  v : any/c
Immediately terminates the current process. Each v argument is ignored.

procedure

(stop π) → void?

  π : process?
Gracefully terminates the execution of π if it is running. If the process is already dead, stop has no effect.

procedure

(kill π) → void?

  π : process?
Immediately terminates the execution of π if it is running. If the process is already dead, kill has no effect.

procedure

(dead? π) → boolean?

  π : process?
Returns #t if π has terminated, #f otherwise.

procedure

(alive? π) → boolean?

  π : process?
Returns #t if π is not dead, #f otherwise.

procedure

(wait π) → void?

  π : process?
Blocks execution of the current process until π has terminated.

procedure

(deadlock v ...) → void?

  v : any/c
Deadlocks the current process. Each v argument is ignored.

procedure

(dead-evt π) → evt?

  π : process?
Returns a synchronizable event that is ready for synchronization if and only if π is dead. The synchronization result of a dead event is the dead event itself.

1.2 Inter-Process Communication

procedure

(give π [v]) → boolean?

  π : process?
  v : msg/c = (void)
Blocks until π is ready to accept the value v on its input channel, or until π is dead. Returns #t if π accepted v, #f otherwise.

procedure

(give-evt π [v]) → evt?

  π : process?
  v : msg/c = (void)
Returns a fresh synchronizable event that becomes ready for synchronization when (give π v) would not block. The synchronization result is the same as the give result.

procedure

(take) → any/c

Blocks until a sender is ready to provide a value on the input channel of the current process. Returns the provided value.

procedure

(try-take) → any/c

Returns the value, if any, provided on the input channel of the current process, or #f immediately if no value is available.

procedure

(take-evt) → evt?

Returns a constant synchronizable event that becomes ready for synchronization when (take) would not block. The synchronization result is the same as the take result.

procedure

(emit [v]) → void?

  v : msg/c = (void)
Blocks until a receiver is ready to accept the value v through the output channel of the current process.

procedure

(emit-evt [v]) → evt?

  v : msg/c = (void)
Returns a fresh synchronizable event that becomes ready for synchronization when (emit v) would not block. The synchronization result is void.

procedure

(recv π) → any

  π : process?
Blocks until π is ready to provide a value through its output channel, or until π is dead. Returns the provided value or eof.

procedure

(recv-evt π) → evt?

  π : process?
Returns a constant synchronizable event that becomes ready for synchronization when (recv π) would not block. The synchronization result is the same as the recv result.

1.3 Control Flow

syntax

(forever body ...)

Repeatedly evaluates bodys forever.

syntax

(while expr body ...)

Repeatedly evaluates bodys for as long as expr evaluates to #t.

syntax

(until expr body ...)

Repeatedly evaluates bodys for as long as expr evalutes to #f.

procedure

(all-evts evt ...) → evt?

  evt : evt?
Returns a single event that is ready for synchronization when all evts are ready for synchronization. The synchronization result is a list of the synchronization results of evts.

procedure

(seq-evts make-evt ...+) → evt?

  make-evt : (-> any/c evt?)
Returns a single event that is ready for synchronization when a series of events is ready for synchronization. Applies the first make-evt to (void) and the remaining make-evts to the synchronization result of the prior event. The synchronization result is the same as the final synchronization result of the event series.

procedure

(loop-evts make-evt ...+) → evt?

  make-evt : (-> any/c evt?)
Returns an event that is never ready for synchronization. The returned event is equivalent to a seq-evts event that loops over make-evts forever.

procedure

(serve proc    
  [#:on-stop on-stop    
  #:command handler]) → process?
  proc : (-> msg/c msg/c)
  on-stop : (-> any) = void
  handler : (-> any) = void
Constructs a server process. Applies proc to each value taken and emits the result.

procedure

(sink proc    
  [#:on-stop on-stop    
  #:command handler]) → process?
  proc : (-> msg/c void?)
  on-stop : (-> any) = void
  handler : (-> any) = void
Constructs a sink process. Applies proc to each value taken and ignores the result.

procedure

(source proc    
  [#:on-stop on-stop    
  #:command handler]) → process?
  proc : (-> any/c)
  on-stop : (-> any) = void
  handler : (-> any) = void
Constructs a source process. Calls proc with no arguments repeatedly and emits each result.

procedure

(socket snk src [#:command handler]) → process?

  snk : process?
  src : process?
  handler : (-> any) = void
Constructs a socket process. Gives what it takes to snk. Emits what it receives from src. Stops snk and src when it stops. Dies when snk or src die.

procedure

(pipe π ... [#:command handler]) → process?

  π : process?
  handler : (-> any) = void
Constructs a pipe process. Gives what it takes to the first π. Gives what it receives from the prior π to the next π. Emits what it takes from the last π. Stops πs when it stops. Dies when any π dies.

procedure

(bridge π1 π2 [#:command handler]) → process?

  π1 : process?
  π2 : process?
  handler : (-> any) = void
Constructs a bridge process. Gives what it receives from π1 to π2 and vice versa. Stops π1 and π2 when it stops. Dies when π1 or π2 die.

procedure

(proxy π [on-take on-emit #:command handler]) → process?

  π : process?
  on-take : (-> msg/c msg/c) = values
  on-emit : (-> msg/c msg/c) = values
  handler : (-> any) = void
Constructs a proxy process. Applies on-take to each value taken and gives the result to π. Applies on-emit to each value emitted by π and emits the result. Stops π when it stops. Dies when π dies.

procedure

(managed π    
  [#:on-eof on-eof    
  #:on-stop on-stop    
  #:command handler]) → process?
  π : process?
  on-eof : (-> any) = stop
  on-stop : (-> any) = void
  handler : (-> any) = void
Constructs a managed process. Applies on-eof to π when it takes or emits eof. Stops π when it stops. Dies when π dies.

procedure

(shutdown π) → void?

  π : process?
Gives eof to π. Blocks until π is dead.

procedure

(shutdown-evt π) → evt?

  π : process?
Returns a fresh synchronizable event that becomes ready for synchronization when (shutdown π) would not block. The synchronization result is void.
